It sounds like you were expecting a simple drag-and-drop or menu option for text formatting, but since GitHub files use Markdown (which is text-based formatting), there isn't a direct "GUI" (Graphical User Interface) button for it.
However, many GitHub platforms, text editors, and IDEs (Integrated Development Environments) that work with Markdown do offer visual buttons or keyboard shortcuts to help you insert the Markdown syntax easily.
Here's how the "GUI" options usually look in an editor:

Summary: The Workflow
Even with GUI buttons, the underlying action is still inserting the special characters (**, *, or `) into your file. The buttons just save you from typing the symbols yourself.
	1. Select the text you want to format.
	2. Click the corresponding icon (like B for bold).
	3. The editor automatically inserts the Markdown syntax around your selected text (e.g., it changes Hello to **Hello**).
